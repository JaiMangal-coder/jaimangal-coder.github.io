1 The Binomial and Negative Binomial Distributions

1.1 The Binomial Distribution

In many introductory courses, we make the simplifying assumption of independence in Bernoulli trials. For example, when playing best two-out-of-three legs in a darts match, we would assume Player A has an 70% chance of winning each leg. We write: \[X \sim \text{Binomial}(n=3, p=0.70)\] where

  • \(X\) is the number of legs won
  • \(n\) is the number of trials
  • \(p\) is the probability of the event of interest

Of note, it is not necessarily the case that the players play all three legs – if either player wins legs one and two the match ends.

There are eight possible outcomes for Player A:

  • (win, win, “win”) \(\longrightarrow\) Player A wins
  • (win, win, “lose”) \(\longrightarrow\) Player A wins
  • (win, lose, win) \(\longrightarrow\) Player A wins
  • (win, lose, lose) \(\longrightarrow\) Player A loses
  • (lose, win, win) \(\longrightarrow\) Player A wins
  • (lose, win, lose) \(\longrightarrow\) Player A loses
  • (lose, lose, “win”) \(\longrightarrow\) Player A loses
  • (lose, lose, “lose”) \(\longrightarrow\) Player A loses

Note: I use quotations above to denote outcomes where the last match isn’t actually played.

Use the Binomial distribution to compute the probability that Player A wins the match. Make sure write out your probability statement and work to write it in terms of the distribution function (PMF/CDF) to plot the probability using ggplot.

Solution

Here, I am looking for \(P(X\geq2)\), the probability that player A wins at least two matches. I can do \[\begin{align*} P(X \geq 2) &= 1 - P(X<2)\\ &= 1 - P(X\leq1)\\ &= 1 - F_X(1)\\ \end{align*}\]

Thus, I can use the CDF function (pbinom) to easily calculate this probability. I’ve also plotted both CDF and PMF below, highlighting the probability we were calculating for.

1.2 The Negative Binomial Distribution

In intermediate courses, we learn the Negative Binomial distribution that we use to model the number of “failures” until the \(r\)th success. Note that this approach still makes the simplifying assumption of independence in Bernoulli trials.

Specify the two cases necessary to compute the probability. Fully explain these probability statements in the context of a best two-out-of-three match and the Negative Binomial distribution. Finally, compute the probability that Player A wins the match and show you get the same result as the Binomial approach. Make sure write out your probability statement and work to write it in terms of the distribution function (PMF/CDF) to plot the probability using ggplot.

Solution

There are two cases for the two out of three match. The first case is player A wins the first two matches in a row, and technically wins immediately. The second case is player A has exactly one loss before their two wins in order to win the whole game. Therefore, let X be the number of failures before Player A gets two wins. So, the probability that player A wins, where X follows a negative binomial distribution, is \(P(X\leq1)\).

\[P(X \leq 1) = F_X(1)\]

I get the same result as I did for the binomial distribution.

2 The Beta Binomial Distribution

One approach to generalize this modeling is to use a Beta-Binomial distribution. This distribution arrises when we let \(p\) be a random variable that follows a Beta distribution: \[\begin{align*} P &\sim \text{Beta}(\alpha, \beta)\\ X &\sim \text{Binomial}(n, P). \end{align*}\] This distribution is not part of base R, but is available using the _bbinom functions from the extraDistr package (Wolodzko 2026).

The primary benefit is that the Beta-Binomial enables us to specify some uncertainty about the skill of Player A, as \(p\) does not have to be fixed (e.g., \(p=0.70\)). Suppose the result of the best two-out-of-three match follows a Beta-Binomial distribution where \(\alpha=7\) and \(\beta=3\).

2.3 The Why

Analytically, we have \[\begin{align*} E(X) &= np &&var(X) = np(1-p) \tag{Binomial Distribution}\\ E(X) &= n\left(\frac{\alpha}{\alpha+\beta}\right) &&var(X) = \frac{n \alpha \beta (\alpha + \beta + n)}{(\alpha+\beta)^2(\alpha + \beta + 1)} \tag{Beta-Binomial Distribution} \end{align*}\]

3 Altham’s Multiplicative Binomial Distribution

While the Beta-Binomial enables us to model cases where winning has a clustering effect (e.g., a “hot hand”).

3.1 Probability Mass Function

The mass function for this distribution is below. While it used to be available in the MBD package for R, this package is no longer available. \[f_X(x) = \frac{1}{c} {n \choose x} p^x (1-p)^{n-x} \theta^{x(n-x)}I(x \in \{0, 1, ..., n\})\] where \[c = \sum_{i=0}^n {n \choose i} p^i (1-p)^{n-i} \theta^{i(n-i)}\]

Write a function in R called daltbinom() for computing the PMF of this distribution. Use your function to plot the distribution where \(p=0.70\) where \(\theta=0.50\), \(\theta=1\), and \(\theta=1.50\). Comment on the differences in the context of a best two-out-of-three match

Solution

Code 
#made my function here
daltbinom = function(x, p, theta, n){
  i = 0:n

  c = sum((choose(n,i))*(p^i)*(1-p)^(n-i)*theta^(i*(n-i)))

  pmf = 1/c * choose(n,x) * p^x * (1-p)^(n-x)*theta^(x*(n-x))

  #use logical for indicator
  return(pmf * (x %in% i))
}

#created my data table
ggdat = tibble(
  x = rep(0:3, times = 3),
  theta = rep(c(0.5,1,1.5),each = 4)
) |>
  mutate(
    PMF = case_when(
      theta == 0.5 ~ daltbinom(x=x, p = 0.7, theta = 0.5, n = 3),
      theta == 1 ~ daltbinom(x = x, p = 0.7, theta = 1, n = 3),
      theta == 1.5 ~ daltbinom(x=x, p = 0.7, theta = 1.5, n = 3)
    )
  )

#plot
PMF.plot = ggplot() +
  geom_linerange(data = ggdat,
                 aes(x = x, ymin = 0, ymax = PMF, color = factor(theta))) +
  geom_hline(yintercept = 0) +
  scale_x_continuous("x", breaks = 0:3) +
  ylab("Probability") +
  theme_bw()+
  facet_wrap(~factor(theta))

PMF.plot

Based on the PMF plot, we see that as we vary theta lower, we get a more ‘hot-handed’ effect, where you’re more likely to win more hands the better you’re doing, so we get a clumpy effect. At theta = 1, we get a normal binomial distribution, as we find there to be no effect on the PMF at that number. At higher thetas, we see a less clumpy effect where the number of legs won is more centered around 2, and thus the hot-hand effect has the opposite effect, where wins result in less wins, and it’s more weighted towards Player A’s leg-win probability, p. Thus, we get more 2-1 games.

3.2 Cumulative Distribution Function

Use your daltbinom() function to create a paltbinom() function for computing the CDF of this distribution. Use your function to plot the distribution where \(p=0.70\) where \(\theta=0.50\), \(\theta=1\), and \(\theta=1.50\).

Solution

Since the CDF is just the probability that X is less than or equal to some value, I can compute it by summing the PMF from 0 up to x.

Code 
paltbinom = function(x, p, theta, n){
  sapply(x, function(z){ #applies function for each 0 to z in x
    sum(daltbinom(x = 0:z, p = p, theta = theta, n = n))}) #sums PMF values from 0 to z
}

ggdat = tibble(

  theta = rep(c(0.5,1,1.5),each = 4)
) |>

      theta == 0.5 ~ paltbinom(x=x, p = 0.7, theta = 0.5, n = 3),
      theta == 1 ~ paltbinom(x = x, p = 0.7, theta = 1, n = 3),
      theta == 1.5 ~ paltbinom(x=x, p = 0.7, theta = 1.5, n = 3)
    )
  )

#plot
CDF.plot = ggplot() +

CDF.plot

Based on the CDF plot we see that changing our theta values affects where the cumulative probability clumps up. At lower theta values, probability clumps reflecting the hot-hand effect. Once again, at theta = 1, we get a normal binomial distribution. At greater theta values, the clumpy effect is reduced, with more probability at middle values, thus more likely heading to 2-1 outcomes.

When theta is 1, the model gives the same result as in the ordinary binomial distribution. When theta is 0.5, the distribution is more clumpy so it is more likely Player A will win as wins influence more wins, already being influenced by Player A’s 70% leg-win chance. When theta is 1.5, distribution is more centerd around the center, thus a greater 2-1 outcome, thus influencing a relatively lower win chance for Player A.

Both the binomial and negative binomial gave a win probability of 0.784, while the beta-binomial model gave a lower win probability of 0.7636. For Altham’s it depends on the theta value used. For lower theta values the win probability is higher than the rest of the models. For higher theta values the win probability is lowered even below all the other models’ win probability. Altham’s model better models the case where legs have a dependent, not independent, effect on other legs.

4 Altham’s Multiplicative Beta-Binomial Distribution

You may have wondered – “Can we do the Beta thing with Altham’s version of the Binomial Distribution?” The answer is yes!

4.1 Probability Mass Function

The mass function for this distribution is below. \[f_X(x) = \left[\int_{0}^1 dp \left(\frac{1}{c} {n \choose x} p^x (1-p)^{n-x} \theta^{x(n-x)}\right)\left(\frac{\Gamma(\alpha+\beta)}{\Gamma(\alpha)\Gamma(\beta)} p^{\alpha-1}(1-p)^{\beta-1}\right)\right]I(x \in \{0, 1, ..., n\})\]

Write a function in R called daltbbinom() for computing the PMF of this distribution. Use your function to plot the distribution where \(\alpha=7\) and \(\beta=3\) where \(\theta=0.50\), \(\theta=1\), and \(\theta=1.50\). Comment on the differences in the context of a best two-out-of-three match.

Hint: You can use the integrate() in R to compute the integral. Note that this requires the function to be vectorized – it should be unless you have added any block conditionals.

Solution

Code 
daltbbinom = function(x, alpha, beta, theta, n){
  #I had an issue because integrate() passes in multiple values of p at once,
  #so I used a second sapply() to evaluate one p-value at a time
  # hence the double sapply, i wont lie i had to consult oscar for this bc i didnt understand why my PMF was not adding up to 1 before. 

  sapply(x, function(z){ #apply from 0 to z in x
    f_x = function(p){
      sapply(p, function(m){ #apply from 0 to m in p (ik my naming convention sucks, i had to try a bunch of things for this to work out)
        daltbinom(x = z, p = m, theta = theta, n = n) * dbeta(m, alpha, beta)
      })
    }
    integrate(f_x, lower = 0, upper = 1)$value
  })
}

ggdat = tibble(
  x = rep(0:3, times = 3),
  theta = rep(c(0.5, 1, 1.5), each = 4)
) |>
  mutate(
    PMF = case_when(
      theta == 0.5 ~ daltbbinom(x = x, alpha = 7, beta = 3, theta = 0.5, n = 3),
      theta == 1 ~ daltbbinom(x = x, alpha = 7, beta = 3, theta = 1, n = 3),
      theta == 1.5 ~ daltbbinom(x = x, alpha = 7, beta = 3, theta = 1.5, n = 3)
    )
  )

PMF.plot = ggplot() +
  geom_linerange(data = ggdat,
                 aes(x = x, ymin = 0, ymax = PMF),
                 linewidth = 1, color = "darkorchid2") +
  geom_hline(yintercept = 0) +
  scale_x_continuous("x", breaks = 0:3) +
  ylab("Probability") +

  facet_wrap(~factor(theta))

When theta is 1.5, the distribution gets to be less clumpy and more centered around 2 wins, results in 2-1 matches more.

The binomial model assumes a fixed leg-win probability of 0.7. So we see a higher trend to 2 and 3 wins. In fact, in all the models it is most probable that Player A’s record is 2-0 in the game. The negative binomial shows the same idea. The beta-binomial is similar but allows the leg-win probability to vary very slightly to adjust for more realistic variations. The Altham Binomial models show how dependencies between legs affect overall outcome. With lower thetas, wins stack and clump together, creating more 2-0 or 0-2 chances, while the opposite is true for higher thetas, creating more 2-1 or 1-2 chances. Combining the Altham with the Beta Binomial we get the combination of leg dependency and variations in leg-win probability. Overall, each model does better with different scenarios. The Altham Beta-Binomial model does well to show hot-hand behavior with varying performance through a match. But, depending on environment and set up, each model will better illustrate those varying systems.
